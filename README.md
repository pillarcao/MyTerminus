# MyTerminus SSH/SFTP 客户端使用手册

## 概述

MyTerminus 是一个基于 Electron + TypeScript 开发的 SSH/SFTP 桌面客户端，支持团队协作场景，可同时使用 SSH 终端和 SFTP 文件传输功能。

## 系统要求

- macOS 10.12+ (ARM64)
- Windows 10+ (x64)

## 安装与使用 (Installation)

### 1. 普通用户直接安装
可以获取预先打包好的安装包直接运行：
- **macOS**: 双击 `MyTerminus-1.0.0-arm64.dmg`，将应用拖入 Applications 文件夹。
- **Windows**: 运行 `MyTerminus-1.0.0-setup.exe` 按照向导完成安装。

### 2. 开发者指南 (Developer Guide)

如果您想从源码运行、修改或构建本机安装包，请遵循如下步骤：

#### 环境依赖
- Node.js (推荐 v18+ 或 v20+)
- npm 

#### 本地启动 (Development)

1. 克隆代码仓库并进入项目根目录：
   ```bash
   git clone <repository_url>
   cd MyTerminus
   ```
2. 安装依赖：
   ```bash
   npm install
   ```
3. 启动开发环境（支持热更新）：
   ```bash
   npm run dev
   ```
   *注意：这会并行启动 Vite 渲染进程与 Electron 主进程。*

#### 生产构建与打包 (Build & Package)

我们使用 `electron-builder` 进行应用程序打包。打包后会在 `dist` (或 `release`) 目录下生成对应平台的可执行文件。

> **⚠️ 跨平台打包注意**：通常建议在目标操作系统上自行打包（在 Mac 上打包 DMG，在 Windows 上打包 EXE）。如果需要在 Mac 上跨平台打包 Windows，需要预先全局安装并配置 `wine`。

1. **基础构建**（只编译前端与主进程，测试代码是否有构建错误）：
   ```bash
   npm run build
   ```

2. **打 macOS 安装包 (.dmg / .app)**：
   如果你当前在 macOS 系统上，请运行：
   ```bash
   npm run package:mac
   # 若环境默认，也可以直接使用 npm run package
   ```
   **输出位置**：完成后会在项目根目录下的特定 release 目录（取决于 `package.json` 中的 `electron-builder` 配置，通常名为 `dist_electron` 或 `build` 等目录）中生成相应的 `.dmg` 安装映像。

3. **打 Windows 安装包 (.exe)**：
   如果你当前在 Windows 系统上，请运行：
   ```bash
   npm run package:win
   ```
   **输出位置**：打包成功后将生成用于 Windows 的 `.exe` 一键安装程序（Setup）。

4. **根据当前系统平台自动打包**：
   ```bash
   npm run package
   ```

## 快速开始

### 1. 创建连接

首次使用时，连接列表为空。点击侧边栏的 **+ New** 按钮创建新连接：

```
┌─────────────────────────────────────────┐
│ Connections                      + New │
├─────────────────────────────────────────┤
│ No connections yet                      │
│                                         │
│ [Add Connection]                        │
└─────────────────────────────────────────┘
```

在弹出的对话框中填写连接信息：

| 字段 | 说明 |
|------|------|
| Connection Name | 连接名称（自定义，便于识别） |
| Host | SSH 服务器地址（IP 或域名） |
| Port | SSH 端口（默认 22） |
| Username | 登录用户名 |
| Authentication | 认证方式：密码 或 私钥 |
| Password | 登录密码 |
| Private Key File | 私钥文件路径（点击 Browse 选择） |

填写完成后点击 **Save** 保存。

### 2. 连接服务器

创建连接后，侧边栏会显示连接列表：

```
┌─────────────────────────────────────────┐
│ Connections                      + New │
├─────────────────────────────────────────┤
│ ▸ My Server                             │
│   root@192.168.1.100:22         [▶]    │
└─────────────────────────────────────────┘
```

点击连接右侧的 **▶** 按钮发起连接。

连接成功后，按钮会变为 **⏹**，并显示 "Connected" 状态。

### 3. 打开终端

连接成功后，会显示两个操作按钮：
- **▸** - 打开 SSH 终端
- **📁** - 打开 SFTP 文件管理器

点击 **▸** 打开终端：

```
┌──────────────────────────────────────────────────────────────┐
│ My Server - Terminal                                    ✕   │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Last login: Mon Apr  7 10:00:00 2026 from 192.168.1.50    │
│  $ _                                                          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

在终端中可以执行各种命令，操作与本地终端一致。

### 4. 使用 SFTP 文件管理

点击 **📁** 打开 SFTP 文件管理器：

```
┌──────────────────────────────────────────────────────────────┐
│ My Server - SFTP                                        ✕   │
├──────────────────────────────────────────────────────────────┤
│ [↻ Refresh] [↑ Up] [↑ Upload] [↓ Download] [+ New Folder]  │
│ /home/username                                               │
├──────────────────────────────────────────────────────────────┤
│ Name          │ Size    │ Modified          │ Actions       │
│ ───────────── │ ─────── │ ───────────────── │ ───────────── │
│ 📁 Documents  │ -       │ 2026-04-01 10:00  │ [🗑]          │
│ 📁 Downloads  │ -       │ 2026-04-02 14:30  │ [🗑]          │
│ 📄 readme.txt │ 1.2 KB  │ 2026-04-03 09:15  │ [🗑]          │
│ 📄 config.json│ 2.5 KB  │ 2026-04-05 16:20  │ [🗑]          │
└──────────────────────────────────────────────────────────────┘
```

#### 导航操作

- **点击文件夹** - 进入该目录
- **↑ Up** - 返回上级目录
- **点击面包屑路径** - 快速跳转到指定目录

#### 文件操作

- **↑ Upload** - 上传本地文件到当前目录
- **↓ Download** - 下载选中的文件到本地
- **+ New Folder** - 在当前目录创建新文件夹
- **🗑 Delete** - 删除文件或文件夹

#### 注意事项

- 删除文件夹时会同时删除其内容
- 上传/下载需要选择本地路径

### 5. 多标签操作

可以同时打开多个终端或 SFTP 标签页：

```
┌──────────────────────────────────────────────────────────────┐
│ My Server - Terminal │ My Server - SFTP                     │
├──────────────────────────────────────────────────────────────┤
│                     ...                                      │
└──────────────────────────────────────────────────────────────┘
```

点击标签页可切换视图，点击 **✕** 可关闭标签页。

### 6. 断开连接

完成操作后，点击侧边栏连接旁边的 **⏹** 按钮断开连接。

或者直接关闭应用，所有连接会自动断开。

## 连接管理

### 编辑连接

点击连接右侧的 **✎** 按钮可修改连接配置。

### 删除连接

点击连接右侧的 **✕** 按钮，系统会确认后删除该连接。

**注意**：删除连接会同时清除保存的认证信息。

## 常见问题

### 连接失败

1. 检查服务器地址和端口是否正确
2. 确认网络连接正常
3. 检查防火墙是否阻止了 SSH 端口
4. 确认用户名和密码/私钥正确

### 终端无响应

1. 尝试按 Enter 键激活
2. 断开重连
3. 检查网络状况

### SFTP 操作失败

1. 确认目标目录有写入权限
2. 检查磁盘空间是否充足
3. 查看错误提示信息
